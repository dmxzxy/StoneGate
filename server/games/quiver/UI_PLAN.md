# QUIVER 界面系统方案（UI_PLAN）

> 目标：把现在"能用但各画各的"界面，整成**一套统一框架 + 一个个清晰子区域**，逐区打磨。
> 方法：先定**全局布局网格**与**通用组件层**(地基) → 再把屏幕拆成命名**子区域** → **逐个**重做完善。
> 约束：480×800 设计空间、触摸优先(命中 ≥ sy(40))、immediate-mode + helium HUD 混合(沿用现状)、零新依赖、每步绿灯门禁(luac+无头冒烟+真机 lovec)可单独验证/回退。

---

## 1. 设计原则 + 设计令牌（统一视觉语言）

- **持久框架 + 模态面板**：顶部状态、底部导航、主舞台常驻；其余都是从框架弹出的模态层，一次只开一个。
- **一眼可读**：每个区域只回答一个问题（我是谁/我在干嘛/我有什么/去哪）。数字带单位、带标签、不堆叠。
- **触摸友好**：可点元素 ≥ sy(40) 高；列表行 ≥ sy(48)；关闭=右上 X + 点遮罩。
- **分类色语言**：稀有度色、活动分组色、元素色固定复用，不另造。

**设计令牌**（落到 `view/ui.lua`，全界面只引这一套）：
```
PAD=sx(12)  GAP=sy(8)  ROW=sy(52)  TAB_H=sy(34)  TITLE_H=sy(40)  TOUCH=sy(44)
R=6  (圆角)   边距/分隔统一用 PAD/GAP
色：复用 D.UI(bg/panel/line/text/dim/good/bad/gold/xp/btn) + 稀有度色 + 元素色
字：font_sm(12 注解) / font(15 正文) / font_med(19 标题) / font_big(30 主数字)
```

## 4. 导航信息架构（IA）重组

现状 8 个目的地散落。重组为 **4 个底部入口**，相关面板用 **tab 合并**：

```
[活动]  → 左抽屉：挂机/战斗/砍柴/采矿/采药/制造/锻造（分组：挂机>战斗>副职业）
[背包]  → 背包面板：tab 全部/材料/箭矢/装备/消耗（高频，独立保留）
[角色▾] → 角色 hub：tab 装备 | 技能 | 精通      （三合一）
[世界▾] → 世界 hub：tab 地区 | 副本             （二合一，副本不再藏地区里）
```
- 制造/锻造是**活动**（选了在主舞台操作），不占导航位；其分类 tab 留主舞台。
- 副本运行/结算时主舞台被副本接管（已是现状），导航在副本中暂隐。
- 好处：顶层从"5 平铺+3 嵌套"降到"4 入口+清晰 tab"，每个 tab 是一个可独立打磨的子区域。

## 5. 通用组件层（地基：新增 `view/ui.lua`，全界面统一）

把各 view 里重复的"面板底/标题/X/返回/tab/滚动列表/物品格/属性行/进度条"收成一套，**所有面板改用它** → 视觉一致、改一处全局生效。immediate-mode，复用 base/draw 原语，零新依赖。

```
ui.frame(opts) -> px,py,pw,ph,bodyY      -- 遮罩+面板底+标题+右上X；anchor: center/left/full
ui.tabs(x,y,w, items, active) -> idx     -- 一行 tab，返回点中索引
ui.scroll(id,x,y,w,h, contentH, drawFn)  -- setScissor+维护偏移的滚动视口
ui.row(x,y,w, sel) ; ui.section(x,y,txt) -- 列表行底 / 分组标题
ui.cell(x,y,s, item, opts) -> {clicked,drag}  -- 物品格(图标+数量+稀有边)
ui.stat(x,y,label,val,col) ; ui.pbar(...)     -- 属性行 / 进度条
ui.btn(id,x,y,w,h,label,col,enabled) -> clicked
ui.close_hit(x,y, px,py,pw) / ui.scrim_hit(...)  -- X/遮罩命中
```
- 命中几何与绘制**共用同一函数**（沿用现有 *_rect 惯例），draw 与 input 不再各算一套。

## 6. 逐区域设计规格（每区目标态，落地时一条条对）

- **R1 顶部状态栏**：左头像卡(等级徽章)；中 HP(粗)/MP(中) 两条；右资源竖排 金币·钥匙·探险许可(带恢复倒计)。**经验移到此处或底部择一**（建议留底部 R4 那条粗的，顶部只 HP/MP，去掉顶部细经验避免重复）。
- **R2 主舞台**：各活动场景统一"上信息(在采什么/在造什么/打谁) + 中演出 + 下进度"三段；战斗加敌人元素抗性提示(弱点/抵抗图标)。
- **R3 技能条**：固定屏底安全区上沿，冷却环 + 法力不足置灰 + 释放白闪；不被场景元素压住。
- **R4 经验条**：Lv + 经验/下一级；满级切"战斗精通 N 点可分配"。
- **R5 底部导航**：4 入口(图标+字)，当前活动色点 + 红点提醒(可学技能/可领副本/背包满)。
- **L 活动抽屉**：补 forge 行；分组标题色条；进行中行高亮 + 职业经验环。
- **O 角色 hub**：tab 装备(纸娃娃+属性卡, DPS 主数字)/技能(已学+技能大师学习)/精通(4 系点数分配)。
- **O 世界 hub**：tab 地区(三档分组可滚 + 推荐度)/副本(列表+推荐等级+掉落预览+许可/钥匙)。
- **O 背包**：顶部筛选 tab(全部/材料/箭/装备/消耗，仅降非选类透明度不重排)；材料按系小标记。
- **O 制造工作台(主舞台内)**：**三轴选择器**(箭头档·元素·翎羽 三排可点) + 成品预览 + 持续制作进度；锻造分类(炼锭/造甲/造弓)同框 tab。
- **F 反馈层**：统一 `fx` 出口(飘字/toast/横幅/粒子)，层级最高，不被面板遮。

## 7. 实施顺序（逐个完善；每步绿灯门禁、独立 commit、可回退）

| 步 | 内容 | 验收 |
|---|---|---|
| **U0 组件层** | 新增 `view/ui.lua`(frame/tabs/scroll/row/cell/stat/pbar/btn + 令牌)；不改现有面板，先建库 + 单元自测 | luac+冒烟绿；ui.lua 各组件可调 |
| **U1 持久框架** | R1 顶部(加钥匙/许可、去顶部重复经验) + R5 底部导航 IA 重组(4 入口) + R4 经验条 | 真机看框架；导航到各 hub |
| **U2 角色 hub** | 装备/技能/精通 合并成 tab 面板(改用 ui.frame/tabs) | 三 tab 切换、原功能不丢 |
| **U3 世界 hub** | 地区/副本 合并 tab；副本入口从"地区里"移到此 | 切区/开副本路径通 |
| **U4 背包 + tooltip** | 筛选 tab + 材料系标记 + tooltip 统一框架 | 72 材料可筛、拖拽不破 |
| **U5 制造/锻造工作台** | 主舞台三轴箭选择器 + 锻造分类同框 | 能可视化选 箭头×元素×翎羽 并造 |
| **U6 主舞台演出 + 技能条** | 战斗/采集场景可读性 + 元素抗性提示 + 技能条固定 | 真机看手感 |
| **U7 反馈层统一 + 全局走查** | fx 出口统一、红点提醒、整体视觉一致性 | 真机整体过一遍 |
